Paperclip 소개

AI 에이전트로
회사를 만들다

6명의 AI 조직이 4일 만에 앱 하나를 완성한 이야기

ulong — 가사 노동 공유 앱 개발기
안녕하세요. 오늘은 Paperclip이라는 플랫폼을 소개하려 합니다. 단순히 AI에게 코드를 짜달라는 게 아니라, AI 에이전트들로 이루어진 가상 회사를 만들고 운영하는 이야기입니다.
"코드를 짜는 AI"가 아니라,
"회사처럼 굴러가는
AI 조직"을 만들 수 있다면?

핵심 컨셉: 사람은 보드(이사회)가 되고, AI 에이전트들이 실무 조직이 된다.

우리가 흔히 알고 있는 "ChatGPT에게 코드 짜달라고 하기"와는 다른 차원의 이야기입니다. 에이전트들이 역할을 가지고 서로 협력하며, 마치 실제 회사처럼 돌아갑니다.
WHAT IS PAPERCLIP

Paperclip이란?

🏢
AI 가상 회사 플랫폼
각 에이전트에게 CEO·CTO·디자이너 등 역할을 부여하고 실제 조직처럼 운영
📋
이슈 기반 작업 관리
이슈(티켓)를 만들어 할당하면 에이전트가 자율적으로 분석·실행·완료
🔄
에이전트 간 협업
위임·리뷰·에스컬레이션이 실제 조직처럼 동작. 크로스팀 협업이 자동으로 이루어짐
💻
로컬 환경 직접 실행
Claude Code 등을 통해 실제 코드를 로컬 환경에서 직접 수정·커밋·빌드
Paperclip은 AI 에이전트들로 구성된 가상 회사를 만들 수 있는 플랫폼입니다. 단순한 챗봇이나 코파일럿과는 다릅니다. 역할이 있고, 책임이 있고, 프로세스가 있습니다.
GETTING STARTED

4단계면 AI 회사가 생긴다

1
회사 이름과 미션 입력
2
첫 에이전트(CEO) 생성 — 어댑터 선택
3
CEO에게 첫 태스크 부여
4
Launch → 에이전트가 즉시 깨어남
회사의 이름을 짓고 미션을 입력한 뒤, CEO 에이전트를 고용하고 첫 업무를 줍니다. 런칭 버튼을 누르면 에이전트가 즉시 깨어나서 업무를 시작합니다.

IN ACTION

이슈 자동 생성

2
CEO가 체크아웃
CEO 에이전트가 이슈를 체크아웃하고 Paperclip 스킬을 로딩해 에이전트 채용 API를 호출합니다.
3
대시보드에서 실시간 확인
1 Agent Running, 1 Task In Progress — 사람은 아무것도 하지 않았습니다.
이것이 실제 동작하는 화면입니다. Launch를 눌렀을 뿐인데, CEO가 알아서 이슈를 체크아웃하고, CTO 에이전트 채용을 시작했습니다. 대시보드 오른쪽에 1 Agent Running, 1 Task In Progress가 보입니다.

HOW IT WORKS

실제 예시
1
ULO-9 "mvp를 만들기 위한 agent 고용"
CEO가 CTO, PM, QAEngineer, Secretary를 순차 채용
2
ULO-16 "ulong MVP React Native 구현"
CTO가 체크아웃하여 앱 전체 구현 완료
3
ULO-18 "MVP 코드 리뷰 및 QA"
QAEngineer가 리뷰 후 버그 목록 작성 → ULO-19로 CTO가 수정
모든 작업은 이슈 단위로 관리됩니다. 보드가 이슈를 만들면, CEO가 분석해서 적절한 에이전트에게 할당합니다. 에이전트는 이슈를 체크아웃해서 작업하고, 완료되면 다음 에이전트가 이어받습니다.

이것이 핵심입니다. 6명의 에이전트가 각자 다른 이슈를 체크아웃하고 동시에 일하고 있습니다. 사람 한 명이 할 수 없는 병렬 작업이 실제로 일어나고 있습니다. 화면 하단에는 6 Agents Enabled, 6 Tasks In Progress가 보입니다.
WHAT WORKED

좋았던 점 👍

⚡
병렬 작업이 가능하다
CTO가 코드를 짜는 동안 PM은 스펙을 쓰고, UX는 디자인한다. 순차 작업 대비 속도가 압도적으로 빠릅니다.
📚
문서화가 자동으로 쌓인다
이슈마다 코멘트·상태·기록이 남습니다. Secretary가 컨벤션 문서와 일지를 자동으로 관리합니다.
🌱
프로세스가 스스로 진화한다
QA 프로세스, 코딩 컨벤션, 디자인 토큰 시스템을 에이전트들이 필요에 따라 자체적으로 만들었습니다.
🎯
사람은 의사결정에만 집중할 수 있다
"이거 만들어줘"라고 이슈 하나 만들면, 에이전트 조직이 분해하고 실행합니다. 방향만 잡으면 됩니다.
가장 인상적인 부분은 병렬 작업입니다. 사람 한 명이 순차적으로 해야 할 일을 여러 에이전트가 동시에 처리합니다. 그리고 문서화가 별도 작업 없이 자동으로 쌓이는 것도 큰 장점입니다.
WHAT NEEDS WORK

아쉬웠던 점 🤔

🌊
컨텍스트 유실
하트비트 단위로 동작해 이전 맥락을 완벽히 이어받지 못할 때가 있습니다. 복잡한 작업일수록 재구성에 시간이 걸립니다.
🚧
블로킹 이슈 해결이 느리다
외부 서비스 설정, 물리 디바이스 테스트 등 블로킹 이슈는 사람이 개입해야 하며, 그 사이 관련 작업들이 멈춥니다.
💬
에이전트 간 소통 비용
이슈 코멘트를 통한 비동기 소통은 체계적이지만, 실시간 논의가 필요할 때 비효율적입니다. 각 에이전트를 깨우는 것 자체가 비용(토큰, 시간)입니다.
🖥️
로컬 환경 의존성
현재 로컬 머신에서 실행됩니다. 리모트 환경 지원이 더 성숙해지면 이 제약은 줄어들 것입니다.
아직 완벽하지는 않습니다. 가장 큰 어려움은 컨텍스트 유실이었습니다. 에이전트가 하트비트 단위로 실행되기 때문에, 이전 작업의 맥락을 완전히 기억하지 못할 때가 있었습니다.
COOL FEATURES

인상적인 기능들 ✨

🔒
체크아웃 시스템
먼저 체크아웃한 에이전트만 작업 가능. Git 브랜치 잠금처럼, 충돌 없이 병렬 작업이 가능한 핵심 메커니즘.
Concurrency Control
💓
하트비트 기반 실행
에이전트는 항상 켜져 있지 않습니다. 이벤트(이슈 할당, 코멘트, 블로커 해제)가 발생할 때만 깨어나 작업하고 종료됩니다. 리소스 효율적.
Event-driven Architecture
🔗
자동 의존성 관리
이슈 간 의존성을 설정하면, 블로커가 모두 해결될 때 자동으로 다음 에이전트를 깨웁니다. "A 끝났으니 B 시작해"를 수동으로 할 필요가 없습니다.
Dependency Resolution
📐
실행 정책
실행 정책으로 리뷰·승인 단계 설정 가능. 코드 작성 → 자동 리뷰어 전달 → 승인 후 다음 단계. CI/CD와 유사한 워크플로우.
Workflow Automation
기술적으로 인상적인 기능들입니다. 체크아웃 시스템은 여러 에이전트가 동시에 작업해도 충돌이 없게 합니다. 하트비트 방식은 리소스를 효율적으로 사용합니다. 에이전트가 24시간 대기하는 게 아니라, 필요할 때만 깨어납니다.
USE CASES

어디에 쓸 수 있나?

🚀
사이드 프로젝트 MVP
혼자서 PM·디자인·개발·QA를 다 해야 할 때. 각 역할을 에이전트에게 위임하면 혼자서도 팀처럼 움직일 수 있습니다.
⚡
빠른 프로토타이핑
아이디어 → PRD → 디자인 → 코드 → QA를 빠르게 순환. 검증까지의 사이클이 극적으로 단축됩니다.
📖
문서화가 중요한 프로젝트
모든 의사결정이 이슈·코멘트로 자동 기록됩니다. 규정 준수가 필요하거나 히스토리 관리가 중요한 환경에 적합.
🔁
반복적인 유지보수
루틴 기능으로 정기 작업을 자동화. 매주 하는 리포팅, 코드 정리, 의존성 업데이트 등을 에이전트에게 위임.
가장 효과적인 상황은 혼자서 팀 전체의 역할을 해야 할 때입니다. 사이드 프로젝트, 프로토타입 개발, 문서화가 중요한 프로젝트에서 특히 유용합니다.
