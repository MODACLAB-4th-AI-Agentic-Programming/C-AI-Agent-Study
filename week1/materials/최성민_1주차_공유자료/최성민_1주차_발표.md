# 하네스 엔지니어링

이 문서는 현재 적용 중인 하네스 엔지니어링에 대한 설명에 대한 문서입니다.

> 핵심: **에이전트 품질을 재현 가능하게 만들기 위한 운영 체계**

---

## 문제점

하네스 없이 에이전트를 사용할 때 자주 발생하는 문제는 아래와 같다.

### 1) 일관성 부족

- 팀/프로젝트의 코드 및 구조 컨벤션이 답변에 안정적으로 반영되지 않는다.

### 2) 최적화 어려움

- 단기적으로 에러를 없애는 수정은 빠르지만, 구조적 해결보다 임시방편으로 문제를 해결하는 경우가 많다.

### 3) 프로젝트 이해 부족

- 프로젝트 목적과 맥락을 충분히 반영하지 못해 의도와 다른 결과를 내놓는 경우가 많다.

### 4) 할루시네이션

- 컨텍스트 누적으로 잘못된 답이 제시되는 경우가 있다.
- 문맥을 부분적으로만 이해했는데도 확신 있게 답하는 경우가 많다.
- 이런 대화가 길어지면 사용자 판단까지 흔들릴 수 있다.

---

## 아이디어

1. 에이전트가 프로젝트 맥락을 먼저 명확히 이해하도록 만든다.
2. 에이전트가 일관된 코드/구조 컨벤션을 따르도록 만든다.
3. 에이전트가 구조적으로 더 나은 해결책을 제안하도록 유도한다.
4. 할루시네이션 가능성을 운영 방식으로 줄인다.

---

## 해결책

### 1) **AGENTS.md**를 요약 + 목차로 설계

[오픈AI 하네스 엔지니어링](https://openai.com/ko-KR/index/harness-engineering/)

Codex는 세션 시작 시 **AGENTS.md**를 읽는다. 따라서 AGENTS.md에 세션의 코어 맥락을 담아, 에이전트가 프로젝트와 컨벤션을 빠르게 이해하도록 만든다.

핵심은 AGENTS.md를 장문 규칙집으로 만들지 않고, **요약 + 목차**로 운영하는 것이다. 상세 내용은 별도 문서에 두고 AGENTS.md에는 위치와 읽기 순서만 명확히 적는다.

AGENTS.md 기본 구성:

1. 프로젝트 요약 설명
2. 세션 시작 시 반드시 읽을 문서와 스킬 목록
3. Always/Never 핵심 규칙
4. 자주 참조하는 문서/스킬의 참조 맵
5. 상황별 적용 스킬/에이전트 목록

### 2) **CONVENTION.md + SKILL.md** 페어 운영

핵심 1. **프로젝트 컨벤션을 에이전트와 사용자가 동일하게 이해**하도록 만든다.

- CONVENTION.md는 구조/코딩/의존 방향/금지사항을 설명하는 기준 문서다.

핵심 2. **컨벤션을 실행 가능한 규칙으로 강제**한다.

- CONVENTION.md를 기반으로 구조 관점/코딩 관점의 필수 규칙을 SKILL.md에 정의한다.
- AGENTS.md의 핵심 규칙에 연결해, 작업 시 해당 스킬이 항상 적용되도록 운영한다.

### 3) 프로젝트 이해와 아이디어 구체화 절차 추가

#### 3-1) PROJECT.md로 공통 맥락 정렬

- PROJECT.md에 프로젝트 목적, 사용자, 핵심 기능, 제약을 정리해 사용자와 에이전트가 같은 전제를 공유한다.

#### 3-2) Superpowers로 요구사항 구체화

- Superpowers는 에이전트가 바로 코드를 작성하지 않고, 요구사항 정리 -> 설계 합의 -> 구현 계획 -> 실행/검토를 체계적으로 수행하게 하는 워크플로우다.

동작 방식:

- 브레인스토밍 단계로 요구사항 구체화
- 짧은 단위의 스펙으로 정리해 사용자 확인
- 승인된 설계를 바탕으로 구현 계획 수립
- 서브에이전트를 활용한 구현 및 검토
- 전체 흐름은 스킬 기반으로 자동 트리거

#### 3-3) 하나의 요구사항 = 하나의 세션

- 위 절차를 적용하면 어떤 세션이든 동일한 프로젝트 이해와 컨벤션이 적용된다.
- 따라서 한 세션에서 한 요구사항만 다뤄 컨텍스트 오염을 줄인다.

---

## 결론

에이전트 개발은 주니어 개발자를 온보딩하고 협업하는 방식과 유사하다고 본 적이 있다.
프로젝트 맥락과 컨벤션을 매번 명확히 전달하고, 일관된 규칙으로 작업을 할 수 있도록 유도하는 과정이 필요하다.

사람이 새 프로젝트에 투입될 때도 설명과 컨벤션 숙지가 필요하듯, 에이전트에게도 같은 정보를 체계적으로 문서화해 제공하는 것이 핵심이다.
