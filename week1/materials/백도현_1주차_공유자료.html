01 / AI Agent

AI 에이전트란?

Agent = 자율적으로 행동하는 주체

AI 에이전트는 환경을 인식(Perceive)하고, 스스로 판단(Reason)하며, 목표 달성을 위해 자율적으로 행동(Act)하는 AI 시스템입니다.

⚡

자율성

사람의 개입 없이 스스로 판단하고 행동

🎯

목표 지향

주어진 목표를 향해 단계적으로 작업 수행

👁

환경 인식

주변 상태를 파악하고 변화에 실시간 적응

🔧

도구 활용

외부 도구와 API를 필요에 따라 호출

💡 전통 AI = 계산기 (시킨 대로만)  →  AI 에이전트 = 비서 (스스로 판단해서 처리)
02 / How It Works

AI 에이전트의 작동 원리

인식 → 판단 → 행동 반복 루프
STEP 1
인식 (Perceive)
환경에서 데이터를
수집하고 상태를 파악
→
STEP 2
판단 (Reason)
수집된 정보를 분석하고
다음 행동을 결정
→
STEP 3
행동 (Act)
도구를 사용하거나
결과를 출력
←————— 반복 (Loop) —————→

이 루프가 반복될수록 에이전트는 이전 결과를 기반으로 더 정확한 판단을 내립니다. 단순 명령 실행이 아닌, '경험'을 통해 점진적으로 목표에 도달하는 것이 핵심입니다.

03 / Skill

스킬(Skill)이란?

에이전트에 부여된 모듈화된 능력 단위

스킬은 에이전트가 수행할 수 있는 특정 기능 단위입니다. 플러그인처럼 필요에 따라 추가/제거 가능한 모듈화된 기능입니다.

💻

코드 작성

프로그래밍 언어로 코드 생성/수정

🔎

웹 검색

인터넷에서 정보를 탐색/수집

📊

데이터 분석

CSV/DB 등 데이터 처리/시각화

📁

파일 관리

파일 읽기/쓰기/수정/삭제

🐛

디버깅

버그를 추적하고 원인 분석/수정

핵심 특징

  • 모듈화 — 독립적으로 추가/제거 가능
  • 재사용 — 여러 에이전트가 같은 스킬 공유
  • 전문화 — 각 스킬은 하나의 역할에 집중
  • 조합 — 여러 스킬 조합으로 복잡한 작업 수행
💡 사람의 '기술' (요리, 운전 등) — 필요한 기술을 배워 능력을 확장하는 것과 동일
04 / Hook

훅(Hook)이란?

특정 이벤트 발생 시 자동 실행되는 트리거

훅은 "~할 때 자동으로 ~해라"는 규칙을 정의하는 이벤트 기반 자동화입니다. 에이전트가 직접 호출하는 게 아니라, 조건 충족 시 자동으로 실행됩니다.

이벤트 발생

파일 저장, 커밋,
도구 호출, 배포 등

→

훅 트리거

조건 확인 후
자동 실행 판단

→

액션 수행

검증, 포맷팅,
알림, 차단 등

🔒

Pre-commit Hook

커밋 전 코드 린트 자동 실행. 규칙 위반 시 커밋 차단.

💾

Pre-save Hook

파일 저장 전 자동 포맷팅. prettier, black 등 실행.

📣

Post-deploy Hook

배포 완료 후 자동 알림. Slack, 이메일 전송.

💡 알람 시계 — 설정한 시간이 되면 무조건 울리는 자동 장치
05 / Skill vs Hook

스킬 vs 훅 비교

구분 스킬 (Skill) 훅 (Hook)
실행 방식 에이전트가 직접 호출 이벤트 발생 시 자동 실행
주체 에이전트 (능동적) 시스템 (수동적 트리거)
목적 작업 수행 능력 확장 품질 보장, 자동화, 안전장치
비유 사람의 기술 (요리, 운전) 알람 시계 (자동 트리거)
예시 코드 작성, 웹 검색, 분석 커밋 전 린트, 저장 시 포맷팅
💡 스킬 = "할 수 있는 것"  |  훅 = "자동으로 일어나는 것"
06 / Multi-Agent

멀티 에이전트 시스템

독립적인 에이전트들이 수평적으로 협력

멀티 에이전트 시스템(MAS)은 독립적인 여러 에이전트가 동등한 위치에서 서로 협력/소통하며 복잡한 작업을 분담하여 해결하는 구조입니다.

코드 작성
에이전트
↔
리뷰
에이전트
↔
테스트
에이전트

수평적 / 동등한 관계 (Peer-to-Peer)

특징

  • 각 에이전트가 독립적으로 판단하고 자율적으로 행동
  • 병렬 처리 가능 — 여러 작업을 동시에 수행
  • 역할 분담 — 전문 영역별 담당 (코딩, 리뷰, 테스트 등)
💡 회사 팀 회의 — 각 팀원이 전문성으로 동등하게 의견을 나누는 구조
07 / Sub-Agent

서브 에이전트

메인 에이전트의 지시를 받아 하위 작업 수행

서브 에이전트는 상위(메인) 에이전트로부터 하위 작업을 위임받아 수행하고, 결과를 다시 상위에게 보고하는 계층적(Parent → Child) 구조입니다.

메인 에이전트 (오케스트레이터)
작업 분해 → 위임 → 결과 통합
↓ 지시 ↓ 지시 ↓ 지시

검색 서브에이전트

↑ 결과 보고

분석 서브에이전트

↑ 결과 보고

작성 서브에이전트

↑ 결과 보고

💡 팀장-팀원 관계 — 팀장이 업무 분배, 팀원이 수행 후 결과 보고
08 / Multi vs Sub

멀티 에이전트 vs 서브 에이전트

핵심 차이점 비교
구분 멀티 에이전트 서브 에이전트
구조 수평적 (Peer-to-Peer) 수직적 (Parent-Child)
관계 동등한 협력 상하 위임
의사결정 각 에이전트가 독립 판단 메인 에이전트가 총괄
통신 에이전트 간 직접 소통 메인을 통한 간접 소통
비유 팀 회의 팀장-팀원
09 / Harness Engineering

하네스 엔지니어링이란?

에이전트 시스템의 구성 요소를 설계하고 조율하는 기술

하네스 엔지니어링은 에이전트, 스킬, 훅을 하나의 체계로 연결하고 조율하는 설계 기술입니다. 어떤 에이전트에게 어떤 스킬을 부여하고, 어떤 훅으로 자동화할지 전체 구조를 설계합니다.

💡 마차의 고삐(harness)가 여러 말을 하나의 방향으로 이끌듯, 하네스가 에이전트/스킬/훅을 하나의 목표로 조율합니다.
①

에이전트 정의

역할/능력/권한을 설계하고 배치

②

스킬 할당

에이전트별 필요한 스킬을 배분

③

훅 설정

이벤트 트리거 규칙을 정의

④

통신 규칙

에이전트 간 소통 방식을 설계

⑤

실행 정책

병렬/순차 전략, 권한과 제한 설정
