# 🤖 #4 AI 에이전트 기반 개발 워크플로우 설계와 실습

모닥랩 4기 AI 에이전트 C조의 스터디 활동을 기록하는 공개 저장소입니다.  
여러 에이전트 개발 기법을 살펴보고, 실무 및 토이 프로젝트에 이를 어떻게 구조적으로 녹여낼지 고민하고 실습합니다.

---

## 🎯 스터디 목표
- **설계-구현-검증**까지 이어지는 AI 에이전트 기반 개발 워크플로우를 구조화합니다.
- 다양한 AI 코딩 에이전트(Claude Code, Cursor 등)를 직접 사용해보고 자신만의 활용 기준을 세웁니다.
- 단순한 개발 결과물보다는 **"에이전트를 활용한 개발의 효율과 방식"** 에 집중하여 인사이트를 공유합니다.

---

## 📅 주차별 학습 계획

| 주차 | 주제 | 상세 내용 |
| :--- | :--- | :--- |
| **1주** | **경험 공유 및 입문** | 실무/토이 프로젝트에 Claude Code, Cursor 등을 적용해본 경험 공유 |
| **2주** | **멀티 에이전트 오케스트레이션** | 여러 AI 에이전트 활용 기법을 적용해보고 상호 작용 방식 탐구 |
| **3주** | **심화 주제 탐색 & 설계** | 관심 있는 심화 주제를 선정하고 4~5주차 토이 프로젝트 설계 |
| **4주** | **토이 프로젝트 진행** | 선정한 주제로 AI 에이전트 활용 프로젝트 수행 (주 1회 이상 진행 상황 공유) |
| **5주** | **결과 공유 & 회고** | 워크플로우 분석, 결과물 공유 및 전체 스터디 회고 |

---

## 📋 운영 방식 및 그라운드룰

> **"모두가 기여하고, 함께 기록하며 성장하는 스터디"** 를 지향합니다.

- **학습 기록**: 학습 과정은 **[Medium](https://medium.com/)** 에 작성하여 기록합니다.
- **모임 기록**: 매주 스터디 진행 내용은 **Slack** 에 공유합니다.
- **적극적 참여**: "모두가 말하는 스터디"를 위해 매 모임 전 **최소 하나 이상의 질문** 을 준비합니다.
- **공유의 원칙**: 자료 공유 시 단순 복붙은 금지하며, 반드시 참조(Reference)를 명시하고 본인의 의견을 덧붙입니다.
- **성실도 관리**: 3주차 1회 이상 결석하거나 그라운드룰을 지속적으로 위반할 경우, 운영진과 긴밀한 면담을 진행합니다.
- **마무리 기록**: 스터디 종료 전, 당일의 인사이트나 소감을 짧게라도 게시합니다.
