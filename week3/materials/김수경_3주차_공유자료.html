01 · Overview

요약

날씨 데이터와 사용자 스타일 프로필을 결합해 오늘의 기본 코디 1세트 + 시간대별 코디를 추천하고, 아이템별 쿠팡 파트너스 제휴 구매 링크를 제공하는 모바일 앱의 Phase 1을 구현한다.

☁️ 날씨 수집

기상청 단기예보 API를 서버에서 30분 캐시로 수집. 앱은 위치 기반으로 날씨와 코디를 서버에서 받는다.

👗 코디 조립

수동 등록 없이 날씨 조건 + 유저 프로필로 검색 키워드를 자동 생성해 쿠팡 파트너스 API 실시간 호출.

💰 수익 구조

쿠팡 파트너스 CPS 모델. 24시간 내 구매 시 수수료(패션 약 3~4%). 제휴 링크는 API 응답에 자동 포함.

02 · Background

배경 & 왜 지금인가

매일 아침 "오늘 뭐 입지?"라는 고민은 10~30대에 공통적으로 발생하지만, 날씨 앱과 패션 앱이 분리되어 있어 두 정보를 연결하는 서비스가 없다.

초기 설계 — 수동 큐레이션 방식 (폐기)

관리자가 코디 세트를 직접 등록하고 날씨 조건에 매핑하는 방식

  • 성별(3) × 연령대(3) × 스타일(6) × 날씨 구간(6) = 최대 324세트 등록 필요
  • 상품 품절·시즌 변경 시 수동 교체 필요
  • 진짜 개인화가 아닌 태그 필터링 수준

채택한 방식 — 키워드 기반 동적 조회

날씨 조건 → 검색 키워드 규칙만 관리자가 정의하고, 실제 상품은 쿠팡 API에서 동적으로 조회 → 운영 부담 최소화 + 실시간 재고 반영 + 진정한 개인화
03 · Goals

목표 (Goals)

  • 앱 실행 시 현재 위치 기반 날씨 (기온, 체감, 강수확률, 바람) 자동 표시
  • 홈 화면에 3시간 단위 기온 타임라인 가로 스크롤 + 탭 시 해당 시간대 코디 포커스 이동
  • 날씨 조건 + 유저 프로필 → 키워드 자동 생성 → 쿠팡 파트너스 API로 상품 조회 → 코디 조립 표시
  • 코디 아이템 클릭 시 인앱 브라우저로 쿠팡 제휴 상품 페이지 이동 (구매 시 수수료 발생)
  • 카카오 / 구글 / 애플 소셜 로그인 + 온보딩(성별, 연령대, 스타일 선택) 후 개인화 코디 추천
  • 관리자 웹에서 날씨 조건 → 키워드 매핑 규칙 등록·수정

07 · Auth

인증 (Passport.js + JWT)

소셜 로그인 플로우

앱 → 카카오/구글 OAuth 페이지
  → authorization code 서버 전달
  → code → access_token 교환 → 유저 정보
  → 신규: DB 저장 + isOnboarded = false
  → JWT 발급 (access 7일 / refresh 30일)
  → 앱: SecureStore에 저장

애플: expo-apple-authentication
  → identityToken → Apple 공개키로 검증
  → 동일 JWT 플로우
애플 로그인 필수 — 앱스토어 가이드라인 4.8: 제3자 소셜 로그인 제공 시 Sign in with Apple 동등 제공 의무. 미포함 시 심사 거절.

JWT 전략

access_token7일
refresh_token30일
저장 위치SecureStore

JWT 갱신

  • access 만료 → refresh로 자동 갱신
  • refresh 만료 → 로그아웃 처리

플랫폼 노트

Apple 로그인은 iOS 전용. expo-apple-authentication 사용. Android에서는 노출 안 함.

  → Redis: "weather:{nx}:{ny}" TTL=35분

2. Redis 날씨 캐시 조회

11 · DB

DB 스키마 (Prisma)

model User {
  id          String   @id @default(cuid())
  provider    String   // "kakao" | "google" | "apple"
  providerId  String
  gender      String?  // "male" | "female" | "none"
  ageGroup    String?  // "10s" | "20s" | "30s"
  styles      String[] // ["casual", "minimal", ...]
  isOnboarded Boolean  @default(false)
  createdAt   DateTime @default(now())
  @@unique([provider, providerId])
}

model KeywordRule {  // 관리자가 등록하는 날씨 조건 → 키워드 매핑
  id       String   @id @default(cuid())
  tmpMin   Int      // 기온 최솟값
  tmpMax   Int      // 기온 최댓값
  ptyType  Int?     // 강수형태 (null=무관, 1=비, 3=눈)
  category String   // "top" | "outer" | "bottom" | "shoes"
  keywords String[] // ["니트", "맨투맨"] — 랜덤 1개 선택
}

model StyleModifier {  // 스타일 → 키워드 수식어 매핑
  id        String   @id @default(cuid())
  styleTag  String   // "casual" | "minimal" | ...
  modifiers String[] // ["캐주얼", "데일리"]
}

14 · Reliability

에러 처리

상황처리 방식
기상청 API 오류 Redis 마지막 정상 데이터 반환. 없으면 "날씨 정보를 불러올 수 없어요" 표시
쿠팡 API 오류 / 결과 없음 키워드 단순화 후 재시도 (수식어 제거). 그래도 없으면 해당 카테고리 제외하고 반환
위치 권한 거부 서울 기본값(37.5665, 126.9780)으로 대체 후 안내 배너 표시
JWT 만료 자동 refresh. 실패 시 로그인 화면으로 이동
15 · Decisions

주요 설계 결정 사항

수동 코디 등록 폐기

성별×연령×스타일×날씨 조합 수백 세트 + 품절 시 수동 교체 → 운영 부담. 날씨→키워드 규칙만 관리하고 상품은 API 동적 조회로 전환.

네이버 쇼핑 수익 없음 확인

초기 "네이버 쇼핑 커넥트"를 수익 채널로 포함했으나, 앱 개발자용 직접 CPS 없음 확인. 수익은 쿠팡 파트너스 단일 채널로 단순화.

Supabase 미사용 결정

벤더 의존성 최소화 + Node.js 직접 제어를 위해 Railway PostgreSQL + Prisma + Passport.js로 분리.

시간대별 코디 & 애플 로그인 Phase 1 포함

일교차 대응이 서비스 핵심 차별점. 애플 로그인은 앱스토어 가이드라인 4.8 의무사항.

코디 결과 캐시 3시간

날씨가 3시간 단위로 변하고, 쿠팡 API 호출 비용 절감. 동일 유저+날씨 조건 결과는 3시간 캐시.
