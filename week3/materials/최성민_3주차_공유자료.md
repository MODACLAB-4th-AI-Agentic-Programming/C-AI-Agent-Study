# 서비스 기획 배경 및 필요성

## 서비스 기획 배경

여행이나 외출을 가고 싶어도, 가장 오래 걸리는 건 "어디를 갈지 정하는 과정"

검색을 시작하면 정보는 많아지지만 결정은 더 어려워지고, 후기/블로그/SNS를 볼수록 비교만 늘고 실제 행동은 늦어짐.

Go-Zip은 이 문제를 줄이기 위한 서비스로, 사용자가 빠르게 하나를 정하고 바로 움직이게 만드는 서비스

### 여행 시장/트랜드의 변화

지도에 펜을 던져 선택된 곳으로 무작위 여행가기 릴스와 같이, 현재 여행의 트렌드는 릴스/숏폼형 여행이 대세가 되는 현상 발생.

장기 계획형 여행보다 반나절 코스, 즉흥 외출, 당일 이동처럼 가볍게 떠나는 흐름이 늘어나는 중.

> **정교한 일정표보다 빠른 결정과 가벼운 시작이 더 중요**

기존 여행 준비 방식은 장소를 검색하고, 후보를 비교하고, 이동 경로와 활동을 조합해 직접 일정을 짜는 구조
이 방식은 꼼꼼한 계획에는 유리하지만, 시간과 에너지가 많이 들고 결국 익숙한 장소만 반복 선택하게 되는 한계가 있음.

반면 지금 사용자가 원하는 것은 복잡한 계획이 아니라 빠르게 실행할 수 있는 하나의 선택지임.

오늘 바로 가볼 곳, 가볍게 움직일 코스, 큰 고민 없이 시작할 수 있는 흐름이 더 중요해지고 있음.

### 현재 문제 상황

- 선택지가 너무 많아서 결정 피로가 큼
- 기존 추천은 인기 장소 중심이라 새로운 경험으로 이어지기 어려움
- 사용자는 추천보다 “지금 뭘 하면 되는지”를 빠르게 알고 싶어 함

## 서비스 필요성

기존 여행 서비스는 검색과 비교에는 강하지만, 막상 아무 데나라도 가고 싶을 때는 약함. 장소를 추천받아도 그다음 동선과 행동은 다시 사용자가 정해야 하기 때문임. Go-Zip은 이 지점을 보완해서, 장소가 아니라 흐름까지 함께 제안하려고 했음.

사용자 입장에서 필요한 이유는 명확함.

- 빠른 결정:
  복잡한 설정 없이 바로 결과 확인 가능
- 새로운 경험:
  익숙한 장소 대신 예상 밖의 코스 제안 가능
- 쉬운 실행:
  장소 목록이 아니라 단계형 흐름으로 제공돼 바로 움직이기 쉬움

# 서비스 개요

## 서비스 소개

Go-Zip은 **즉시 출발할 수 있는 랜덤 외출/여행 시나리오 추천 서비스**

즉, “어디 가지?”를 오래 고민하게 만드는 대신, “이렇게 움직여 보자”라는 실행 계획을 바로 보여주는 방식

사용자는 모드를 고르고, 가벼운 조건만 선택하는 것으로, 이동, 지역, 행동을 조합해 바로 출발 가능한 결과를 만들어 시나리오 생성

- 결과가 빠르게 생성되는지
- 사용자가 빠르게 결정할 수 있는지
- 결과가 바로 실행 가능한지

## 주요 기능

### 1. 4가지 랜덤 생성 모드

Go-Zip은 목적에 따라 4개 모드를 제공함.

- `여행모드`
  지역과 행동을 묶어 짧은 탐방 코스 제안
- `신속모드`
  지금 바로 움직일 수 있는 빠른 외출 시나리오 생성
- `거지모드`
  최대 예산 안에서 도보 중심의 저비용 코스 생성
- `랜덤모드`
  이동, 음식, 행동, 미션을 강하게 섞은 완전 랜덤 시나리오 생성

### 2. 가벼운 조건 선택

- 여행모드: 이동 수단, 추천 방식
- 신속모드: 이동 수단, 이동 강도
- 거지모드: 최대 예산
- 랜덤모드: 입력 없이 바로 생성

### 3. 룰 기반 시나리오 생성 엔진

전국 시·군·구 단위 지역 데이터, 이동 수단, 행동 카테고리, 음식 데이터, 예산 조건을 조합해 결과를 생성

현재 기준으로는 버스, 지하철, 도보, 자동차, 자전거 등 5가지 이동 수단을 반영하고 있으며,
여기에 지역 태그와 행동 편향까지 함께 적용해서 결과가 너무 단순한 랜덤으로 보이지 않도록 설정

- 신속모드: 즉시 실행 가능한 행동 우선
- 거지모드: 예산과 도보 이동 우선
- 여행모드: 여행 분위기와 복수 행동 코스 강화

### 4. 결과 화면

결과는 단순한 장소 리스트가 아니라, 그대로 따라갈 수 있는 단계형 흐름으로 제공

사용자는 요약 문장, 이동 방식, 도착지, 행동 정보, 단계 리스트를 보면서 바로 행동을 진행

즉, 결과를 본 뒤 다시 계획을 세우는 것이 아니라, 결과 자체가 하나의 실행 계획이 되도록 파이프라인형태 구성

예시1:

1. 버스타고 14 정거장 이동
2. 도착지에서 500m 도보 이동
3. 카페에서 커피 한 잔
4. 근처 공원에서 산책하기

예시2:

1. 충청남도 태안군
2. 자동차로만 이동하기
3. 해산물 맛집에서 점심 먹기
4. 해변에서 일몰 감상하기

### 5. 재생성, 저장, 가이드

결과가 마음에 들지 않으면 같은 조건 안에서 다시 생성 가능함. 결과 단계 리스트는 이미지로 저장 가능함. 또 가이드 화면에서 4개 모드의 차이와 사용법을 따로 설명해 처음 사용하는 사람도 쉽게 이해할 수 있게 했음.

## 서비스 차별성

- 장소 추천보다 시나리오 제안에 가까움
- 결과를 받은 뒤 다시 고민하지 않아도 됨
- 모바일 중심의 짧은 흐름으로 빠르게 생성 가능함
- 랜덤을 4개 모드로 나눠 목적에 맞게 경험을 세분화했음

정리하면, 더 잘 추천하는 서비스보다 더 빨리 움직이게 만드는 서비스

# 서비스 발전 방향 (MVP 이후, 현재 미구현)

## 개발 서비스 발전 방향

### 기능 확장

- AI 기능 추가:
  결과 문장 고도화, 추천 이유 설명 강화, AI와 연동하여 더욱 세밀한 여행 시나리오 추천
- 오픈 API 연동:
  지도, 대중교통, 날씨, 지역 행사, 운영 시간 반영

### 서비스 고도화

- 개인화:
  재생성 패턴, 선호 수단, 예산대 반영으로 결과 품질 향상
- 커뮤니티:
  다녀온 시나리오 공유, 챌린지·랭킹으로 참여 동기 확장
- 장기 방향:
  단순 생성 도구가 아닌 즉흥 외출 중심 플랫폼으로 확장

### 수익화

서비스 흐름을 해치지 않는 방식으로 접근함.

- 광고:
  지역 콘텐츠·여행 서비스 연계 맥락형 광고
- 제휴:
  카페, 식당, 숙소, 액티비티 혜택 연결
- 프리미엄:
  세밀한 조건 제어, 개인화 강화, 나만의 랜덤 코스 제공
